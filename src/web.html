Context

Ce outil a été créé dans le but de faciliter l'extraction les données de modèles ainsi que celles des observations. Ce programme est écrit en Python 3.4 avec des scripts pour le system UNIX en TCL/tk et [[Bash|BASH]].

Procédure à suivre pour installer l'outil

1.1 Allez dans le répertoire /space/hall1/sitestore/eccc/oth/airq_central/sair001/Ding_Ma/projetQA-StartUpFiles pour télécharger l'outil

1.2 Copiez le ficher setup.[[Bash|bash]] et files.zip dans votre répertoire de travail

1.3 Exécutez le script de [[Bash|bash]]. Si le scipt de functionne pas, il n'est pas en exécutable. Faite la command "chmod 744 setup.[[Bash|bash]]" dans votre terminal

1.4 Redémarez votre terminal pour utilisez l'outil

1.5 Utilisez la commande AIDE pour démarer l'outil

L'autre manière est de faire cd ~VotreRepoTravail et par la suite faire python3 -m UI.py

Comment Utiliser l'Outil?

Premiere section -- Rentré des données

[[Image:P1.png|thumb|alt=XRARC|Première Section|right|600px]]

Il faut rentrer les informations dans tous les boîtes et clickez sur "write to file". Si vous faites un changement, il faut repeser sur le boutton vert "update" pour faire un mise à jour des configurations. La partie de l'extraction des archives se fait avec la série de date rentrée. Par contre, le traitement se fait jour par jour.

Deuxième section -- Selection des modèles

[[Image:P2.png|thumb|alt=XRARC|Deuxième Section|right|600px]]

Choisiez les modèles dont vous voulez les informations. Pour les Gemmach, FireWork, et Umos-Mist, il faut choisir la date. Les UMOS et observations se font automatiquement sur la list de date.

  • Le premier boutton réécrit le fichier standard avec un script [[Bash|bash]] pour isoler les heures et polluants selectionés
  • Le deuxième effectue les calculs avec [[SPI|SPI]] pour interpoller les resultats à la station choisi

Si les fichiers ont déjà été traités par le script [[Bash|bash]], il suffit de faire un mise à jour des fichiers avec la nouvelle station pour obtenir des résultats ailleur.

Pour les UMOS et Observation, il suffit simplement d'extraire les données des archives et de selectionés la station intéressée.

Troisième section -- Images

[[Image:P3.png|thumb|alt=XRARC|Troisième Section|right|600px]]

Pour extraire les images des archives, il faut toujours Utiliser les données entré dans la section 1. Deplus, il faut aussi selectionés les partie de la carte à la droite.

Modèles Gem-Mach

  • Le premier boutton extrait les donnes des archives en utilisant la command rarc
  • Le deuxième génère l'animation gif dans le répertoire de sortie

Modèles UMOS-Mist

  • Le premier boutton extrait les données des archives en utilisant la commande rarc
  • Le deuxième génère l'animation gif dans le répertoire de sortie. Par contre, les images UMOS contiennet deux types ([[Surface|surface]] ou différence de [[Surface|surface]]), il faut aussi choisir ceci avant de générer l'image

En bas au complet, le boutton 'Animate GIF' est utilisé pour jouer le ficher choisi. Ce boutton utlise la fonction animate du système linux.

Suprimer les Fichiers Brutte

Le boutton "Delete Extracted Files", va suprimer les répertoires des archives dans ~/rarc/. Ces fichiers sont extrêmement grand, environ 1GBé/h pour les Gemmach et Firework. C'est important de les suprimer quand on a finit avec. C'est normal que le programme lance l'errur "Exception in Tkinter callback" lorsqu'on suprime le répertoire, c'est un bug avec Python.

Liste des Fichiers Modifiables par l'Utilisateur en cas de Besoin

Le fichier stations.csv est configuré de cette façons: numéro de station,nom englais,nom français, lat,lon,province (en numéro, consultez le fichier info.csv pour la conversion),ville,address.

Layout des fichiers Python----------

Le fichier UI.py est le ficher root du programme. Il importe tout les autre fichiers Python. Chaque fichier est indépendant des autres, elles sont tous des modules séparés. C'est pour cela que becaucoup de code se resemble. Pour ajourter des modules dans le future, il faut simplement l'importer dans UI.py et adapter le code.

Erreur ou Bug Possible

  • Le programme peut seulement prendre 2 mois au maximum
  • Une boite doit être choché au minimum pour les modèle, les particules, et les location
  • Dans la case de "search", il faut seulement entrer des charactères(nom de station) ou des nombres(code de station).
  • Si rarc retourne "Grand Total of 0", ceci veut dire que les fichiers d'archives n'existent pas ou elles sont déjà présentes.
  • Les dates avant le 7 avirl 2016 pour les UMOS ne vont pas marché dû à des erreurs dans les fichiers standard
  • Parfois, quand les fichiers d'observations viennent de se faire extraire, les données ne sortent des bd. Il faut peser sur "update" une fois.

[[Bash|Bash]] Script

Pour le script de [[Bash|bash]], le pgramme va isoler les heures et polluants. Ceci génère un script [[Bash|bash]] comme FireWork00.[[Bash|bash]], il y a deux fichiers par modèles pour les deux passes. Le debut du script est la configuration pour le script. Ceci inclus l'heure, les polluants, les niveaux, et la passe. Le script prend le ficher brute des archives et il va le combiner en 1 seul ficher de sortie .fst. Le fichier de sortie est situer à ~/[[Bash|bash]]/. Dépendant du type de modèle, le ficher de sortie aurait cette forme MODEL-MODÈLE_DATE-DÉBUT_DATE-FIN.fst. Dans ce script, la commande DESIRE est utilisé. Ceci va chercher les tic-tacs ainsi que les polluants au niveaux désirer.

Note: Le script pour les Umos-Mist contiennent aussi la commande ZAP, ceci permet de changer l'ETICKET à CAPAMIST car les tic-tacs ne le sont pas avant une certaine [[Date|date]]. De plus, les fichiers Umos-Mist ne sont pas accessible avant le 7 avril 2016 car le timestamp a des erreurs.

Extraire aux stations

Pour l'extration des données à une station, le script extract1.tcl et le fichier de configuration sont nécessaires. Le programme Python génère un fichier config par heure, par [[Date|date]], par passe, par polluants et par station. Ceci fait becoup de fichier! Ceci évite des problèmes que le script TCL peut avoir. Si le temps spécifié n'est pas dans le fichier de sortie du script [[Bash|bash]], le script TCL ne va pas fonctionner. C'est pour ça qu'il a des erreur dans le script tcl roule (error 8), c'est normal! À la fin, le script va générer becaucoup de ficher csv avec une entré. Ceux qui sont vide sont suprimer par le programme et ceux qui contiennent de l'information sont combiner ensemble. Le répertoire de sortie se situe à ~/output/. Les fichiers de configuration se situent à ~/config/ et les fichier de traitement se situent à ~/extracted/. Bien évidament, il y a une répertoire par modèle.

Les scripts sont lancés avec la commande ./extract1.tcl ~/CONFIG.tcl, les fichiers config sont nommés dans cette convention: MOIS-PARTICULE-JOUR-HEUR-PASSE.tcl. Elles sont suprimées au début de la prochaine utilisation.

Observations

Pour les observations, l'extraction des archives est la même que tous les autres, mais pour obtenir les résultats des stations, il faut utiliser des commande SQL. Les fichiers sont archivés en bloc de 3h. Les données des stations sont placées dans un style de key:value dans le fichier database. L'outil va premièrement chercher les heures intéressée, trouve la "clé". Avec la clé, on peut ciblé les valeurs (dans ce cas la measure des polluants).

Umos

Il y a deux façons de extraire les umos. Avant 2017, les umos sont archivés par polluant. Donc, il suffit de les déarchiver et d'utiliser la commadne cat et grep avec la station et les polluants pour obtenir les résultats. Après cette [[Date|date]], il faut d'abord utiliser la commande cmcarc, pour déarchiver les polluant intéressée et par la suite c'est avec la commande cat et grep pour obtenir les résultats.

ETICKET

Le programme génère des petits scripts MODÈLE-Eticket.tcl, pour obtenir les eticket des fichier standard, ceci évite l'utilisateur de les rentrées manuellement. Les scripts sont automatiquement adaptés pour les dates. Si le ficher contient deux ETICKET, le programme ne va pas fonctionner.

Images, GIF

Les images sont déarchiver avec la commande cmcarc. Ceci va sortir les 48 fichiers de la passe du modèle. Par la suite, la commande convert -delay 35 -loop ~/imgTemp/ combine tout les images pour former une animation. Le Gem-Mach et Umos fonctionne de la même manière. Le fichier de sortie se situe a ~/output/ avec le nom MODELE_AN-MOIS-[[Date|DATE]]-HEURE_POLLUANT_ENDROIT.gif. Les Umos contiennent @sfc ([[Surface|surface]]) ou @sfc@diff(différence de [[Surface|surface]]) après l'endroit. Pour les jouer, la commande unix de animate est utilisée
